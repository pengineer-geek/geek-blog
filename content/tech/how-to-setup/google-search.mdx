## はじめに

「書いたのに検索に出ない…」は誰もが通る道。  

このブログ（**Vercel + Next.js + `*.vercel.app`**）も、公開直後は**どのキーワードでも無反応**でした。ここでは**実際に検索に出るまで**にやったことを、**タイムライン＋手順**で公開します。

---

## 🔑 手順のざっくり流れ（全体像）

1. **Search Console にアクセス & ログイン**
2. **プロパティを追加（今回は *URLプレフィックス* を選択）**
3. **所有権を確認（メタタグ or HTML ファイル）**
4. **サイトマップ（sitemap.xml）を登録**
5. **各記事のインデックス申請（URL検査）**
6. **数時間〜1日待ち → 反映確認**

> **なぜ URLプレフィックス？**  
> 今回は `https://pengineer-geek-blog.vercel.app` を運用していたため、**URLプレフィックス**で登録。  
> ※将来、独自ドメインを導入してサブドメイン含め一括で管理したいなら**ドメインプロパティ**が便利（DNS認証が必要）。

### Day 0（そのまま続行）
- **所有権の確認方法**は **メタタグ**を選択。  
  `layout.tsx` の `<head>` に verification メタを追加 → デプロイ。

```tsx
// src/app/layout.tsx
import type { Metadata } from "next";
import "./globals.css";

export const metadata: Metadata = {
  title: "サイトのタイトル",
  description: "サイト説明",
};

export default function RootLayout({ children }: { children: React.ReactNode }) {
  return (
    <html lang="ja">
      <head>
        {/* ▼ GSC 所有権確認（URLプレフィックスの場合） */}
        <meta name="google-site-verification" content="xxxxxxxxxxxxxxxxxxxx" />
      </head>
      <body>{children}</body>
    </html>
  );
}
```

> 代替案：**HTMLファイル方式**
> `public/googlexxxxxxxxxxxxxxx.html` を置いてもOK（GSCで指示されるファイル名と中身をそのまま配置）。

## うまくいかなかった時に見たポイント（実体験）

* **URL が 404 のまま**：
  デプロイ反映前に申請してた。**デプロイ完了後**に再申請でOK。
* **サイトマップの URL が間違い**：
  `*.vercel.app` と独自ドメインを混在 → **運用ドメイン側に統一**。
* **`noindex` がテンプレに残ってた**：
  下書き時の名残。**本番ビルドの `<meta name="robots">` を確認**。
* **内部リンクが薄くてクロール遅い**：
  トップ・カテゴリ・関連記事から**内部リンクで辿れる**ように調整。

---

## まとめ（体験談の所感）

* **やったことはシンプル**：GSC（URLプレフィックス）登録 → 所有権確認 → サイトマップ → URL検査
* **最初の反映は1日程度**。その後も順位は**数日〜数週間で上下**。
* 反映スピードを上げたいなら、**内部リンク**と**サイト構造の一貫性**が効く。
* そして何より、**更新を続ける**のが最強（ここがいちばん難しい…！）

---

## 付録：運用メモ（Next.js）

* **記事の `updated`** をちゃんと更新 → `sitemap.xml` の `lastModified` が効く
* **OG画像/タイトル**は毎回確認（SNS拡散→被リンクの入口になりやすい）
* **`/posts/[...slug]`** の生成で 404 が混ざるとクロールの無駄が増えるので注意
